# CSV 파일 분석: 파트별 택배 수령지 취합_기준정보.csv

## 파일 구조

### 헤더
```
NO,기사명,사번,사용자,파트,기사코드,배송지
```

### 컬럼 설명
| 컬럼명 | 설명 | 데이터 예시 |
|--------|------|------------|
| NO | 순번 | 1, 2, 3... |
| 기사명 | 차량/기사 식별명 | FL72, FL73, FL26 등 |
| 사번 | 사원번호 | S19063, 217253, 218395 등 |
| 사용자 | 사용자 이름 | 정지훈, 윤순규, 정다민 등 |
| 파트 | 소속 파트/팀 | 외주, TM센터_FL지역팀 1파트, TM센터_FL수도권팀 2파트 등 |
| 기사코드 | 기사 코드 | 9100007, 9100225, 9100169 등 |
| 배송지 | 배송지 주소/명칭 | 경동화물 화성남양안석451 영업소, 전남 장성군 황룡면 월평리 609-3 AJ렌탈 등 |

## 데이터 특징

### 1. 파트 구조
- **외주**: 외주 기사들
- **TM센터_FL지역팀**: 1파트, 2파트, 3파트, 4파트
- **TM센터_FL수도권팀**: 1파트, 2파트, 3파트, 4파트

### 2. 배송지 정보
- 대부분 "경동화물 [지역] 영업소" 형식
- 일부는 "AJ렌탈 [지역]센터" 형식
- 일부는 상세 주소 형식
- 빈 배송지도 존재 (배송지 정보가 없는 경우)

### 3. 사용자 정보
- 사번이 사용자ID로 사용 가능
- 기사코드는 9100000 형식의 숫자
- 사용자 이름은 한글 이름

## 시스템 매핑 전략

### 사용자 데이터 매핑
```javascript
{
  사용자ID: 사번 (예: S19063, 217253)
  이름: 사용자 (예: 정지훈, 윤순규)
  기사코드: 기사코드 (예: 9100007)
  소속팀: 파트 (예: 외주, TM센터_FL지역팀 1파트)
  지역: 파트에서 추출 (예: 지역팀 → 지역 추출, 수도권팀 → 수도권)
  역할: '신청자' (기본값)
  활성화: 'Y' (기본값)
}
```

### 배송지 데이터 매핑
```javascript
{
  배송지명: 배송지 (예: 경동화물 화성남양안석451 영업소)
  소속팀: 파트 (예: 외주, TM센터_FL지역팀 1파트)
  주소: 배송지 (상세 주소인 경우)
  활성화: 'Y' (기본값)
}
```

## 데이터 처리 로직

1. **사용자 등록**
   - 사번을 사용자ID로 사용
   - 중복 체크 (이미 등록된 사용자는 스킵)
   - 기본 비밀번호: 1234

2. **배송지 등록**
   - 파트별로 배송지 그룹화
   - 배송지가 비어있지 않은 경우만 등록
   - 중복 배송지 체크 (같은 파트, 같은 배송지명)

3. **지역 추출**
   - 파트명에서 지역 정보 추출
   - "지역팀" → 지역 정보 필요
   - "수도권팀" → "수도권" 또는 파트별 지역 매핑

## 주의사항

1. 일부 행은 배송지만 있고 사용자 정보가 없는 경우 (빈 행)
2. 배송지가 비어있는 경우도 존재
3. 파트명이 길고 복잡함 (예: "TM센터_FL지역팀 1파트")
4. 지역 정보가 파트명에 직접 포함되지 않아 별도 매핑 필요
