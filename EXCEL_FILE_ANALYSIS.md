# Excel 파일 분석: 파트별 택배 수령지 취합_기준정보.xlsx

## 파일 정보
- **파일명**: 파트별 택배 수령지 취합_기준정보.xlsx
- **목적**: 기준정보 데이터 (배송지, 사용자 등)

## 예상 구조 분석

### 1. 배송지 정보 (Delivery Places)
파일명에 "파트별 택배 수령지"가 포함되어 있으므로, 파트(팀)별 배송지 정보가 포함되어 있을 것으로 예상됩니다.

**예상 컬럼 구조:**
| 컬럼명 | 설명 | 필수 여부 |
|--------|------|----------|
| 배송지명 | 배송지 이름 (예: 서울영업소, 부산영업소) | ✅ 필수 |
| 소속팀 | 파트/팀 이름 (예: 서울1팀, 부산1팀, 본사) | ✅ 필수 |
| 주소 | 배송지 주소 | ❌ 선택 |
| 연락처 | 배송지 연락처 | ❌ 선택 |
| 담당자 | 배송지 담당자 이름 | ❌ 선택 |
| 활성화 | 사용 여부 (Y/N) | ❌ 선택 (기본값: Y) |
| 비고 | 기타 메모 | ❌ 선택 |

**예상 데이터 예시:**
```
배송지명        | 소속팀    | 주소                    | 연락처        | 담당자 | 활성화 | 비고
서울영업소      | 서울1팀   | 서울시 강남구...        | 02-1234-5678 | 홍길동 | Y      |
부산영업소      | 부산1팀   | 부산시 해운대구...      | 051-1234-5678| 김철수 | Y      |
본사            | 본사      | 충남 천안시...          | 041-1234-5678| 박영희 | Y      |
```

### 2. 사용자 정보 (Users)
기준정보 파일이므로 사용자 정보도 포함되어 있을 수 있습니다.

**예상 컬럼 구조:**
| 컬럼명 | 설명 | 필수 여부 |
|--------|------|----------|
| 사용자ID | 로그인 ID | ✅ 필수 |
| 이름 | 사용자 이름 | ✅ 필수 |
| 기사코드 | 기사 코드 | ❌ 선택 |
| 소속팀 | 소속 팀/파트 | ❌ 선택 |
| 지역 | 지역 (서울, 부산 등) | ❌ 선택 |
| 역할 | 신청자/관리자 | ❌ 선택 (기본값: 신청자) |
| 활성화 | 사용 여부 (Y/N) | ❌ 선택 (기본값: Y) |

### 3. 시트 구조 예상
Excel 파일이 여러 시트로 구성되어 있을 가능성이 있습니다:

1. **배송지관리** 시트 (또는 "배송지", "수령지", "택배" 등)
2. **사용자관리** 시트 (또는 "사용자", "직원" 등)
3. 기타 기준정보 시트

## 데이터 매핑 전략

### 배송지 데이터 매핑
```javascript
// Excel 헤더 → 시스템 헤더 매핑
{
  '배송지명' / '수령지명' / '택배지명' → '배송지명',
  '소속팀' / '파트' / '팀' → '소속팀',
  '주소' / 'Address' → '주소',
  '연락처' / '전화번호' / 'Contact' → '연락처',
  '담당자' / 'Manager' → '담당자',
  '활성화' / 'Active' / '사용여부' → '활성화',
  '비고' / 'Remarks' / 'Memo' → '비고'
}
```

### 사용자 데이터 매핑
```javascript
// Excel 헤더 → 시스템 헤더 매핑
{
  '사용자ID' / 'ID' / '아이디' / '이메일' → '사용자ID',
  '이름' / 'Name' → '이름',
  '기사코드' / 'Employee Code' → '기사코드',
  '소속팀' / 'Team' / '파트' → '소속팀',
  '지역' / 'Region' → '지역',
  '역할' / 'Role' → '역할',
  '활성화' / 'Active' / '사용여부' → '활성화'
}
```

## 다음 단계

1. **Excel 파일을 Google Sheets로 변환**
   - Google Drive에 Excel 파일 업로드
   - "파일 열기" → "Google 스프레드시트로 변환"

2. **데이터 구조 확인**
   - 각 시트의 헤더 확인
   - 데이터 형식 확인
   - 필수 필드 확인

3. **임포트 함수 개발**
   - 시트별 데이터 읽기
   - 헤더 매핑 (유연한 매핑 로직)
   - 데이터 검증 및 저장

4. **테스트**
   - 샘플 데이터로 테스트
   - 에러 처리 확인
   - 중복 데이터 처리 확인

## 참고사항

- Excel 파일은 직접 읽을 수 없으므로 Google Sheets로 변환 필요
- 또는 CSV로 변환하여 처리 가능
- 파일 구조가 예상과 다를 수 있으므로 유연한 매핑 로직 필요
